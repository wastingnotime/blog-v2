# Slice: 2026-04-17 Robots Txt Bootstrap

## Selected Pack

- `python_ddd_monolith`

## Runtime Targets

- Python build runtime
- static browser runtime
- filesystem-backed publication output

## Architecture Mode

Deterministic crawler-policy publication for the static site:

- emit one bounded `robots.txt` artifact during the build
- derive the sitemap reference from the configured static-site base URL
- keep the policy static and compatible with GitHub Pages hosting

## Discovery Scope

The restored publication now emits human-facing pages plus machine-readable
artifacts such as `feed.xml`, `sitemap.xml`, `search.json`, RSS autodiscovery,
identity assets, and Open Graph metadata. One publication-facing contract is
still missing from the generated output: `robots.txt`.

That gap is already visible in repository artifacts:

- `docs/slices/2026-04-16-publication-metadata-bootstrap.md` explicitly scoped
  `robots.txt` out of the earlier metadata slice
- the current generated `dist/` artifact set does not contain `robots.txt`

This slice restores the minimum crawler-policy surface needed for the current
static publication:

- generate a single `robots.txt` file at build time
- allow crawling of the public site surface by default
- point crawlers to the generated sitemap using the configured base URL

This slice does not attempt route-specific crawl controls, environment-specific
policies, alternate user-agent sections, or broader SEO strategy work.

## Use-Case Contract

### `BuildRobotsPolicy`

Given site settings and the generated publication contract, generate
deterministic static output for `/robots.txt` such that:

- the artifact exists in the build output without any runtime dependency
- the default policy allows crawling of the public static site
- the file advertises the canonical sitemap URL derived from `SITE_BASE_URL`
- output stays deterministic for the same repository state and configuration

### `ProjectRobotsPolicy`

Given site settings, compute deterministic crawler-policy lines for:

- a single wildcard user-agent section
- a bounded allow/disallow policy for the current public site
- one sitemap declaration using the configured base URL

## Main Business Rules

- `robots.txt` must be generated from repository-owned configuration rather than
  maintained as a handwritten output artifact in `dist/`.
- The bootstrap policy remains deliberately permissive: `User-agent: *` plus no
  disallow rules that would hide the public site.
- The sitemap declaration must resolve to the generated `sitemap.xml` artifact
  using the configured static-site base URL.
- The slice stays bounded to one shared crawler policy for the current static
  publication.
- GitHub Pages compatibility remains a hard constraint.

## Required Ports

- environment reader for site settings
- filesystem writer for static build artifacts
- existing publication metadata contract for the sitemap route

## Initial Test Plan

Run the scenario CLI against the in-repo content set and inspect generated
artifacts to verify:

- `robots.txt` exists in `dist/`
- the file contains a wildcard user-agent section
- the file points to the generated `sitemap.xml` artifact using the configured
  canonical base URL
- the publication stays fully static and free of application-server assumptions

## Done Criteria

- the build emits a deterministic `/robots.txt` artifact
- the generated policy allows the current public site surface to be crawled
- the sitemap declaration uses the configured canonical base URL
- deterministic tests cover generated output and unchanged static-only behavior
