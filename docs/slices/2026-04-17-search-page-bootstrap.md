# Slice: 2026-04-17 Search Page Bootstrap

## Selected Pack

- `python_ddd_monolith`

## Runtime Targets

- Python build runtime
- static browser runtime
- filesystem-backed publication output

## Architecture Mode

Static search entry surface backed by the existing search index:

- publish one generated `/search/` page under the shared site frame
- point readers to the already-generated `search.json` artifact
- keep the slice bounded to a lightweight client-side discovery surface

## Discovery Scope

The restored publication now emits `search.json`, archives, topic pages, saga
navigation, and machine-readable metadata, but it still lacks a dedicated route
where readers can actually search the site.

That gap is explicit in current artifacts:

- `docs/slices/2026-04-16-search-index-bootstrap.md` restored the static index
  but explicitly deferred the search page and live filtering UI
- the current generated `dist/` output has `search.json` but no `/search/`
  route

This slice restores the minimum search surface needed for the current static
publication:

- generate one `/search/` page at build time
- include a bounded client-side search interface that loads `search.json`
- render deterministic empty-state content when JavaScript has not filtered
  results yet

This slice does not attempt server-side search, fuzzy ranking, multi-index
faceting, or an advanced application-like search experience.

## Use-Case Contract

### `BuildSearchPage`

Given site settings and the shared publication frame, generate deterministic
static output for `/search/` such that:

- the page exists in the build output without any backend dependency
- the page points to the generated `search.json` artifact
- readers can search the publication through client-side filtering
- output remains compatible with GitHub Pages hosting

### `RenderSearchInterface`

Given the static publication contract, render bounded search-page behavior such
that:

- the page provides an input for searching published content
- the page displays deterministic initial guidance before a query is entered
- result links resolve to existing static routes under the configured base URL

## Main Business Rules

- The search page must use the existing static `search.json` artifact rather
  than inventing a new search backend.
- The bootstrap search interface stays intentionally lightweight and
  deterministic.
- Search results should cover the published reading surface already represented
  in `search.json`.
- The slice stays bounded to one shared search route rather than broader client
  application state.
- GitHub Pages compatibility remains a hard constraint.

## Required Ports

- shared HTML document renderer for generated routes
- existing `search.json` publication artifact
- filesystem writer for static build artifacts
- environment reader for site settings
