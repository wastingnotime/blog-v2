# Slice: 2026-04-17 Archives Bootstrap

## Selected Pack

- `python_ddd_monolith`

## Runtime Targets

- Python build runtime
- static browser runtime
- filesystem-backed publication output

## Architecture Mode

Deterministic archive projection for the static publication:

- project a chronological archive from repository-authored content
- publish one generated archive route under the shared site frame
- keep the slice bounded to static listing behavior rather than search or
  faceted discovery

## Discovery Scope

The restored publication now exposes homepage editorial framing, saga and topic
navigation, machine-readable metadata, search records, Open Graph tags, and a
root crawler policy. One reader-facing navigation surface is still missing:
an archive page for chronological browsing across published entries.

That gap is visible in current repository artifacts:

- earlier slice docs explicitly deferred archive views as non-goals while
  restoring homepage, taxonomy, and saga navigation
- the current generated `dist/` output contains no archive route

This slice restores the minimum archive surface needed for the current static
publication:

- generate one `/archives/` page at build time
- list standalone pages and episodes in deterministic reverse-chronological
  order
- keep links compatible with static hosting and existing shared chrome

This slice does not attempt year/month partitioning, pagination, faceted
filtering, full-text search, or a broader redesign of the information
architecture.

## Use-Case Contract

### `ProjectArchiveIndex`

Given the content catalog and existing entry metadata, project archive rows such
that:

- entries are ordered by publication date descending
- each row exposes title, permalink, kind, summary, and lightweight narrative
  context when available
- results remain deterministic for the same repository state

### `BuildArchivePage`

Given site settings, projected archive rows, and the shared publication frame,
generate deterministic static output for `/archives/` such that:

- the page exists in the build output without any runtime dependency
- readers can browse published entries chronologically from newest to oldest
- generated links resolve correctly under the configured static-site base URL
- the page remains compatible with GitHub Pages hosting

## Main Business Rules

- The archive page must be derived from repository-owned content metadata, not
  maintained as a handwritten page.
- Archive ordering stays strictly reverse-chronological for the bootstrap slice.
- Structural routes such as section hubs are excluded; the archive lists
  published reading entries only.
- Entry context may include saga and arc labels when those values can be
  projected deterministically from the existing catalog.
- The slice stays bounded to one shared archive surface rather than introducing
  pagination or alternate grouping schemes.
- GitHub Pages compatibility remains a hard constraint.

## Required Ports

- content loader output for repository-authored content
- shared HTML document renderer for generated routes
- filesystem writer for static build artifacts
- environment reader for site settings
